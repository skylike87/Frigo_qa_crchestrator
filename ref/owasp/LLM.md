## Extracted Needed Parts (Project-Applied)

### Prompt & Output Safety (LLM01/LLM02)
- Treat all prompt inputs and retrieved context as untrusted.
- Apply strict output validation before using model output in app logic/UI.
- Block instruction override patterns and high-risk tool calls.
- Add indirect prompt injection pre-filters for external sources (PDF/web/DB text)
  before context insertion (e.g., hidden override directives).
- When chaining agents, treat upstream agent output as untrusted input and re-scan it
  before passing to downstream agents.

### Data Protection (LLM03/LLM06)
- Prevent sensitive local/app/user data from being exposed to prompts/logs.
- Minimize prompt payloads and redact secrets/PII before model invocation.
- Validate RAG source integrity/provenance and write permissions for indexed content.
- Enforce log masking for API keys, tokens, credentials, and personal data in LLM traces.

### Runtime Abuse Controls (LLM04/LLM08)
- Enforce token/latency/cost budgets and concurrency limits.
- Restrict model “agency” (tool/action permissions) by explicit allowlist.
- Require schema/type/range validation for every tool/function parameter generated by LLM.
- Prohibit direct execution patterns such as `eval()`/shell passthrough from raw model output.
- Run tool execution in sandboxed runtime (container/isolated worker) with constrained resources.
- Apply least-privilege separation per agent role in multi-agent workflows.

### Supply Chain & Trust (LLM05/LLM10)
- Track model/provider/version dependencies and update risk.
- Protect model access paths and API credentials from extraction abuse.

### Human-in-the-loop (LLM09)
- Mark AI output as assistive, not authoritative.
- Add confirmation gates for high-impact actions.

## Immediate Test Checklist for This Repo
- [ ] Prompt construction path reviewed for injection opportunities.
- [ ] External source ingestion (PDF/web/DB) includes indirect prompt-injection detection.
- [ ] Agent-to-agent handoff payloads are re-validated/sanitized before reuse.
- [ ] Output parsing path reviewed for unsafe execution/use.
- [ ] Tool/function input parameters are schema-validated (type/range/allowlist).
- [ ] No raw model output is directly executed by interpreter/system command.
- [ ] Sensitive fields redaction verified pre/post LLM call.
- [ ] LLM logs mask API keys/tokens/PII consistently.
- [ ] User session/context isolation is enforced so prompts/RAG memory never cross tenants.
- [ ] LLM request limits and fallback behavior validated.
- [ ] Model refusal responses are handled safely (no crash/unsafe fallback; clear user guidance).
- [ ] Tool/action permissions reviewed for least privilege.
- [ ] Multi-agent role permissions are separated by least privilege.
- [ ] User-facing AI decisions require explicit confirmation where needed.
